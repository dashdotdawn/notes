        1. 回流重绘 DOM tree + style strcut = render tree render tree 只包括可视部分（不含 display:none, head)

        reflow: 需要重新验证 render tree 的局部或全部，重新计算节点大小，至少发生一次（初始化页面布局时），也称作 Layout repaint: 屏幕的部分去要需要进行更新，可能因为节点的几何结构、样式改变。指元素的样式改变不影响文档流整体结构，渲染树结构不变。

        触发 Layout/paint/composite 的 css 属性

        浏览器会将 reflow 动作放入队列延迟并合并执行，获取某些样式时，浏览器为了返回准确的值会导致强制回流，例如

        offsetTop/Left/Width/Height
scrollTop/Left/Width/Height
clientTop/Left/Width/Height
getComputedStyle / currentStyle(IE)
widht, height

        频繁的强制回流会造成布局抖动，应该尽可能避免：

        • 静态样式维护 class 而不是具体样式
        • 动态样式作多次操作时应缓存局部变量，避免过度操作计算
        • 离线处理多个 DOM 操作，例如 documentFragment；克隆节点上更新后替换；display: none 操作完毕后恢复显示

        可以使用 devtool 中的 performance 来查看性能瓶颈，红色小三角一般是需要注意的 warning 部分。

        优化

        1. 减少回流，避免或减少强制回流
        2. 兼容性允许时使用 flex

        避免大型、复杂的布局和布局抖动
