在使用数组方法的时候需要注意的一点是方法是否改变原数组, 会更改原数组的方法有：sort | reverse | push pop unshift shift | splice；map | filter | slice | 会返回新数组。

方法 注意事项
join 不指定分隔符默认使用逗号
concat 参数为数组时则连接数组元素，如 [1].concat(2, [3, [4, 5]]) = [1,2,3,[4,5]]
splice 可以修改原数组，返回被删除元素组成的数组
toString 将每个元素转化为字符串后用逗号分隔输出，如 [1,2,[3,[4,5]]].toString() = [1,2,3,4,5]
every, some every 任意 some 存在，返回 boolean，空数组的 every 为 true，some 为 false
indexOf, lastIndexOf 可以指定第二个参数，表示开始搜索的位置
reduce 不指定初始值时使用第一个元素作为初始值，空数组不带初始值调用 reduce 将导致类型错误异常，单个元素数组将直接返回该元素

小哥：forEach 在遍历的时候会跳过 null、undefined 值

真的是这样吗？

[null].forEach(a => console.log(a))
>null

其实并不完全是这样的，forEach 和 map 在遍历的时候会跳过的是稀疏数组的空位，而不是存在的 undefined 或 null 值。

var arr = []
arr[1] = 3
arr.push(undefined)

// arr 就是一个稀疏数组
0 in arr // false
1 in arr // true
2 in arr // true

虽然 arr[0] === arr[2] 都是 undefined，但是对于数组来说，0 就是一个空位，2 则是真实存的的 undefined 值，因此 for...in、forEach、map 都只会遍历到 1、2。

数组是一种特殊的对象，可以使用非负整数作为索引，索引相当于一种特殊类型的属性名。而作为对象，当然可以拥有任意的属性名，因此 JavaScript 中的数组不存在越界。

arr[-1.1] = 1 // "-1.1" 会作为属性名
arr['1'] = 1 // 相当于 a[1] = 1
arr[1.0] = 1 // 相当于 a[1] = 1

最后写一下解

var data = [null, {name: 'a'}, {name: 'b'}, {name: 'a'}, null]

function uniq(arr) {
  var uniq_arr = []
  var names = {}
  for (var i = 0; i < arr.length; i++) {
    let name = arr[i] && arr[i].name
    if (!names[name]) {
      names[name] = 1
      uniq_arr.push(arr[i])
    }
  }
  return uniq_arr
}

uniq(data)

# 数组遍历方式的区别

方法 特性
for for (var i = 0;i < arr.length; i++)
forEach, map 相当于在上一条的 for 循环中增加了 i in arr 的判断
for...in 遍历一个对象的可枚举属性，包括继承属性，不保证顺序；因此不建议用来遍历数组
for...of 遍历所有数据结构的统一方法，如 Set\Map\arguments\NodeList\数组\字符串等，依赖于 Symbol.iterator 属性；用来遍历数组时与第一种 for 循环效果一致，只不过取到的是值而不是索引
// 一种去掉数组奇数位的方式，蛮有趣的
var list = [1,2,3,4,5,6]
list.forEach((a,i,arr) => {
  arr.splice(i, 1)
})
